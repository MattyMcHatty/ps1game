COMMITTING CHANGES
------------------
Always stage and commit files from these folders when they have changed
or when new files have been added to them:

    assets/          — level meshes (.smd), prop meshes, other game assets
    blender models/  — source .blend files for any 3D models
    music/           — background music tracks (.wav, .mp3, etc.)
    sounds/          — sound effects (.vag, .wav, etc.)
    src/             — all C source files (.c) and headers (.h)
    textures/        — texture files (.tim, .png, source images)

Also always commit these specific files if they have changed:
    CMakeLists.txt          — build configuration
    disc.xml                — disc layout (controls what goes on the disc)
    system.cnf              — PS1 boot config
    rebuild commands.txt    — build instructions
    git_details.txt         — this file
    .gitignore              — controls what git ignores

When committing, stage everything relevant at once rather than file by file.
Example full commit workflow:

    git add src/ textures/ assets/ sounds/ music/ CMakeLists.txt disc.xml
    git commit -m "Your description of what changed and why"
    git push

To also pick up NEW files in those folders (git add stages new files too):
    git add src/ textures/ assets/ sounds/ music/ "blender models/"
    git add CMakeLists.txt disc.xml system.cnf "rebuild commands.txt" git_details.txt .gitignore
    git commit -m "Description"
    git push


WHAT IS INTENTIONALLY IGNORED (.gitignore)
-------------------------------------------
These are excluded from git and should NOT be committed:

    build/       — all compiled output (HORROR.exe, HORROR2.BIN, object files)
    *.exe        — compiled ELF binaries
    *.bin/*.BIN  — raw PS-EXE and disc images (large, always re-generated)
    *.cue        — disc cue sheets (re-generated by mkpsxiso)

The disc image (build\HORROR.bin + build\HORROR.cue) is always rebuilt
locally using mkpsxiso — it does not need to be in version control.
